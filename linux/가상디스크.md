# Loop / RAM Disk I/O 정리

## Loop 장치 (Loopback Device)

- `.iso`, `.img` 파일을 블록 디바이스처럼 마운트할 때 사용
- **블록 레이어를 거침** → iostat에 `loop0`, `loop1` 등으로 잡힘

### 생성/제거

```bash
# 이미지 생성 및 마운트
dd if=/dev/zero of=test.img bs=1M count=100
mkfs.ext4 test.img
sudo mount -o loop test.img /mnt/test

# 제거
sudo umount /mnt/test
sudo losetup -d /dev/loop0
```

## RAM Disk

### 종류별 비교

| 종류 | 블록 디바이스 | iostat |
|------|------------|--------|
| `tmpfs` | ❌ | ❌ |
| `ramfs` | ❌ | ❌ |
| `/dev/ram0` (brd) | ✅ | ✅ |

- `tmpfs` / `ramfs` → 블록 레이어 bypass, VFS에서 페이지 캐시로 바로 접근 → iostat에 안 잡힘
- `oflag=direct` (O_DIRECT) 는 tmpfs에서 지원 안 함 → `Invalid argument`

### brd 생성/제거

```bash
# 생성
sudo modprobe brd rd_nr=1 rd_size=131072
sudo mkfs.ext4 /dev/ram0
sudo mount /dev/ram0 /mnt/ramdisk

# 제거
sudo umount /mnt/ramdisk
sudo rmmod brd
```

## Loop I/O가 sda에 안 잡히는 이유

```
dd → loop0 (iostat에 잡힘)
         ↓
   .img 파일의 page cache에 write
         ↓
   writeback 스레드가 나중에 sda로 flush
```

- loop I/O와 sda writeback은 **타이밍이 분리됨**
- writeback은 커널 파라미터 조건에 따라 **지연/분산**되어 나감

### 관련 커널 파라미터

```bash
sysctl vm.dirty_background_ratio    # = 10  (메모리 10% 초과 시 백그라운드 writeback)
sysctl vm.dirty_expire_centisecs    # = 3000 (30초 지난 dirty 페이지 flush)
sysctl vm.dirty_writeback_centisecs # = 500  (5초마다 writeback 스레드 실행)
```

### 50MB가 sda에 안 보이는 이유

- `dirty_background_ratio = 10` → 메모리 4GB 기준 400MB 이상이어야 트리거
- 50MB는 조건 미달 → 비율 트리거 안 걸림
- 30초에 걸쳐 5초 간격으로 조금씩 분산 → 초당 수 kB 수준 → 배경 노이즈에 묻힘
- `sync`로 강제 flush 해도 sda에 안 보임 (미해결)

### dirty 모니터링

```bash
watch -n 0.5 'grep Dirty /proc/meminfo'
```

## sync

- dirty 페이지(메모리에만 있고 디스크에 안 내려간 데이터)를 **즉시 강제 flush**
- 안 쳐도 커널이 알아서 writeback하지만 타이밍은 커널 재량

```bash
sync
```
