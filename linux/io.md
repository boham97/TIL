# 랜덤 I/O vs 순차 I/O

디스크에서 데이터를 읽고 쓰는 두 가지 방식. 인프라 성능 이해의 핵심 개념.

## 순차 I/O (Sequential I/O)

디스크의 **연속된 위치**를 차례대로 읽거나 쓰는 방식.

### 예시

- 큰 동영상 파일을 처음부터 끝까지 읽기
- 로그 파일을 끝에 계속 추가하기
- 백업 작업

### 특징

- HDD에서 헤드가 한 번 자리잡으면 계속 그 자리에서 읽기 가능
- **매우 빠름**

## 랜덤 I/O (Random I/O)

디스크의 **흩어진 위치**를 그때그때 옮겨다니며 읽고 쓰는 방식.

### 예시

- DB 인덱스를 따라 흩어진 레코드 가져오기
- 트랜잭션 처리

### 특징

- HDD에서 헤드가 매번 다른 위치로 이동(seek) + 회전 대기 필요
- **매우 느림**

## 성능 차이 (HDD 기준)

|구분       |시간/속도     |비고                                 |
|---------|----------|-----------------------------------|
|랜덤 액세스 1회|5~10ms    |대부분이 seek time + rotational latency|
|순차 읽기    |100MB/s 이상|헤드 이동 없음                           |

→ 순차 I/O가 랜덤 I/O보다 **수십~수백 배** 빠를 수 있음

## SSD에서의 차이

- 물리적 헤드 이동이 없어서 랜덤 I/O 페널티가 훨씬 작음
- 그래도 순차가 더 빠름 (컨트롤러가 큰 블록을 한 번에 처리하는 게 효율적)
- 격차는 HDD처럼 극단적이지는 않음 (**2~5배 정도**)

## 병목 관점에서의 의미

### DB 설계

- 인덱스는 랜덤 I/O를 일으킴
- 풀 스캔(순차 읽기)이 오히려 빠를 때도 있음

### 워크로드별 권장 스토리지

|워크로드   |특성       |추천       |
|-------|---------|---------|
|로그·백업  |순차 쓰기 위주 |HDD로도 충분 |
|트랜잭션 DB|랜덤 액세스 많음|SSD가 큰 효과|

### OS·DB의 최적화 전략

- **캐시**: 자주 쓰이는 데이터를 메모리에 보관
- **미리 읽기(prefetch)**: 랜덤을 순차처럼 변환(c fseek)
