일반적인 네트워크 스택의 헤더/푸터
 (pcap은 L2 헤더~payload만 담고 L1 프레이밍과 FCS는 대개 생략)

  ---
  전체 그림 — 선 위를 흐르는 실제 프레임

  │◄── L1 물리 프레이밍 ──►│◄──────── pcap이 담는 범위 ────────►│◄ L1 ►│
  ┌──────────┬─────┬────────┬─────────┬─────────┬──────────┬─────┬──────┐
  │ Preamble │ SFD │ Eth 헤더│ IP 헤더 │TCP/UDP  │ payload  │ FCS │ IFG  │
  │   7B     │ 1B  │  14B   │ 20~60B  │ 20/8B   │          │ 4B  │ 12B  │
  └──────────┴─────┴────────┴─────────┴─────────┴──────────┴─────┴──────┘
    └─ 헤더 ─┘              └────────── 헤더 ──────────┘        └푸터┘

  ---
  L1 — 물리 프레이밍 (pcap엔 없음)

  ┌─────────────────────────────┬──────┬────────────┬───────────────────────────────────────────────────┐
  │            요소             │ 크기 │ 헤더/푸터  │                       의미                        │
  ├─────────────────────────────┼──────┼────────────┼───────────────────────────────────────────────────┤
  │ Preamble                    │ 7B   │ 헤더       │ 10101010... 반복. 수신 NIC이 클럭 동기화하는 신호 │
  ├─────────────────────────────┼──────┼────────────┼───────────────────────────────────────────────────┤
  │ SFD (Start Frame Delimiter) │ 1B   │ 헤더       │ 10101011. "여기부터 진짜 프레임" 시작 표시        │
  ├─────────────────────────────┼──────┼────────────┼───────────────────────────────────────────────────┤
  │ IFG (Inter-Frame Gap)       │ 12B  │ 푸터(간격) │ 프레임 사이 최소 유휴 시간. 실제론 침묵 구간      │
  └─────────────────────────────┴──────┴────────────┴───────────────────────────────────────────────────┘

  → NIC 하드웨어가 처리하고 소프트웨어엔 안 올라오므로 pcap/코드엔 없습니다.

  ---
  L2 — Ethernet (헤더 14B + 푸터 4B)

  ┌──────────────────┬──────┬────────────────────────────────────────────────────────────┐
  │       필드       │ 크기 │                            의미                            │
  ├──────────────────┼──────┼────────────────────────────────────────────────────────────┤
  │ 목적지 MAC       │ 6B   │ 받는 NIC 하드웨어 주소                                     │
  ├──────────────────┼──────┼────────────────────────────────────────────────────────────┤
  │ 출발지 MAC       │ 6B   │ 보낸 NIC 하드웨어 주소                                     │
  ├──────────────────┼──────┼────────────────────────────────────────────────────────────┤
  │ EtherType/Length │ 2B   │ 상위 프로토콜 (0x0800=IPv4, 0x86DD=IPv6)                   │
  ├──────────────────┼──────┼────────────────────────────────────────────────────────────┤
  │ (VLAN 802.1Q)    │ +4B  │ 옵션. VLAN ID + 우선순위                                   │
  ├──────────────────┼──────┼────────────────────────────────────────────────────────────┤
  │ FCS (푸터)       │ 4B   │ CRC-32 체크섬. 프레임 전송 중 비트 오류 검출 → 틀리면 폐기 │
  └──────────────────┴──────┴────────────────────────────────────────────────────────────┘

  👉 네트워크에서 "진짜 푸터"라 부를 만한 건 이 FCS가 거의 유일합니다.

  ---
  L3 — IP (푸터 없음)

  IPv4 (20~60B, 가변)

  ┌──────────────────────┬───────┬───────────────────────────────┐
  │         필드         │ 크기  │             의미              │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ Version/IHL          │ 1B    │ 버전 + 헤더 길이(가변 원인)   │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ DSCP/ECN             │ 1B    │ QoS 우선순위, 혼잡 신호       │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ Total Length         │ 2B    │ IP 패킷 전체 길이             │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ ID/Flags/Frag Offset │ 4B    │ 단편화 재조립 정보            │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ TTL                  │ 1B    │ 홉 제한 (루프 방지)           │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ Protocol             │ 1B    │ 상위 프로토콜 (6=TCP, 17=UDP) │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ Header Checksum      │ 2B    │ 헤더 무결성                   │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ 출발/목적지 IP       │ 8B    │ 논리 주소                     │
  ├──────────────────────┼───────┼───────────────────────────────┤
  │ (Options)            │ 0~40B │ 옵션 (거의 안 씀)             │
  └──────────────────────┴───────┴───────────────────────────────┘

  IPv6 (40B, 고정)

  - IPv4보다 헤더가 크지만 고정 길이(옵션은 확장 헤더로 분리) → 라우터 처리가 빠름. 체크섬 없음(L2/L4에 위임),
  단편화 필드 없음(끝단만 단편화).

  ---
  L4 — 전송 (푸터 없음)

  UDP (8B, 고정) — 우리 프로젝트

  ┌──────────────────┬──────┬───────────────┐
  │       필드       │ 크기 │     의미      │
  ├──────────────────┼──────┼───────────────┤
  │ 출발/목적지 포트 │ 4B   │ 프로세스 구분 │
  ├──────────────────┼──────┼───────────────┤
  │ Length           │ 2B   │ 헤더+payload  │
  ├──────────────────┼──────┼───────────────┤
  │ Checksum         │ 2B   │ 무결성(옵션)  │
  └──────────────────┴──────┴───────────────┘
``
  L2 FCS(이더넷 CRC)가 이미 링크 구간 비트 오류를 잡아주기 때문에, 단일 링크/로컬 환경에서는 UDP 체크섬이 중복이라고 보고 끄는 경우가 있습니다.
저지연이 중요한 시스템(예: 프로젝트에서 쓰는 IEX-TP 같은 마켓데이터 피드)은 체크섬 계산 비용조차 아끼려고 SO_NO_CHECK 같은 소켓 옵션으로 꺼버리기도 합니다.
다만 이 경우 라우터 내부 메모리 비트플립처럼 L2 FCS로 못 잡는 손상은 그대로 애플리케이션까지 올라올 수 있다는 트레이드오프가 있습니다.
```

  TCP (20~60B, 가변) — 일반적 경우

  ┌───────────────────┬───────┬───────────────────────────────┐
  │       필드        │ 크기  │             의미              │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ 출발/목적지 포트  │ 4B    │ 프로세스 구분                 │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Sequence Number   │ 4B    │ 바이트 순서 (순서보장·재조립) │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Ack Number        │ 4B    │ 수신 확인                     │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Data Offset/Flags │ 2B    │ 헤더 길이 + SYN/ACK/FIN 등    │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Window Size       │ 2B    │ 흐름 제어(받을 수 있는 양)    │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Checksum          │ 2B    │ 무결성                        │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ Urgent Pointer    │ 2B    │ 긴급 데이터                   │
  ├───────────────────┼───────┼───────────────────────────────┤
  │ (Options)         │ 0~40B │ MSS, SACK, Timestamp 등       │
  └───────────────────┴───────┴───────────────────────────────┘

  → UDP는 "그냥 보냄", TCP는 순서·재전송·흐름제어를 헤더로 다 관리. 이게 UDP 8B vs TCP 20B+ 차이의 이유입니다.

  ---
  L7 — 응용 (프로토콜마다 다름, 일부는 트레일러 있음)

  응용 계층엔 표준 헤더/푸터가 없고 프로토콜마다 자유지만, 대표 예:

  ┌──────────────────┬─────────────────────┬────────────────────────────────────────────────────┐
  │     프로토콜     │        헤더         │                   푸터/트레일러                    │
  ├──────────────────┼─────────────────────┼────────────────────────────────────────────────────┤
  │ TLS (record)     │ 5B (타입/버전/길이) │ MAC 또는 AEAD 인증 태그 (16B 등) ← 무결성 트레일러 │
  ├──────────────────┼─────────────────────┼────────────────────────────────────────────────────┤
  │ HTTP             │ 텍스트 헤더 (가변)  │ 없음 (\r\n\r\n로 구분)                             │
  ├──────────────────┼─────────────────────┼────────────────────────────────────────────────────┤
  │ IEX-TP (우리 것) │ 40B (세션/시퀀스)   │ 없음                                               │
  └──────────────────┴─────────────────────┴────────────────────────────────────────────────────┘

  👉 TLS의 인증 태그가 L7에서 "푸터"의 대표 사례입니다. 암호화 데이터 뒤에 붙어 변조를 검출합니다.

  ---
  한눈에 — 헤더/푸터 크기 요약

  ┌──────┬──────────────┬─────────────────┬─────────────────┐
  │ 계층 │   프로토콜   │      헤더       │      푸터       │
  ├──────┼──────────────┼─────────────────┼─────────────────┤
  │ L1   │ Ethernet PHY │ Preamble+SFD 8B │ IFG 12B(간격)   │
  ├──────┼──────────────┼─────────────────┼─────────────────┤
  │ L2   │ Ethernet     │ 14B(+VLAN 4B)   │ FCS 4B          │
  ├──────┼──────────────┼─────────────────┼─────────────────┤
  │ L3   │ IPv4 / IPv6  │ 20~60B / 40B    │ 없음            │
  ├──────┼──────────────┼─────────────────┼─────────────────┤
  │ L4   │ TCP / UDP    │ 20~60B / 8B     │ 없음            │
  ├──────┼──────────────┼─────────────────┼─────────────────┤
  │ L7   │ TLS / IEX-TP │ 5B / 40B        │ 인증태그 / 없음 │
  └──────┴──────────────┴─────────────────┴─────────────────┘

  기억할 3가지

  1. "진짜 푸터"는 L2 FCS(CRC 4B)가 사실상 유일 — 하위 계층에서 전송 오류를 잡기 위한 트레일러. (L7 TLS 인증
  태그는 예외적 케이스)
  2. 헤더는 아래로 갈수록 "물리적", 위로 갈수록 "논리적" — MAC(하드웨어) → IP(논리 주소) → 포트(프로세스) →
  앱(세션/데이터).
  3. 가변 길이 헤더는 IPv4·TCP (옵션 때문). IPv6·UDP·Ethernet은 고정이라 하드웨어 파싱이 빠름 — 저지연 설계가
  UDP를 쓰는 이유와 통합니다.
