1. 프로젝트 루트 디렉토리에서 클로드코드실행하기
- 이동하는데 토큰 많이 듬

2. /init으로 클로드.md 파일 만들기
- 클로드가 모든 파일을 읽고 계획이나 테스크 실행함 -> 토큰 소모량 큼
- 클로드.md에 파일구조를 정리해 일부만 보도록 함
- 절대 하면 안되는것 명시하기
    - main에 푸시X
    - db에 쿼리X
- 코딩 컨벤션 (카멜 케이스,..)
- 핵심패턴 등
- 아키텍쳐 (트리형식)
- 기술스택
- 빌드 테스트
- 도메인 컨텍스트 (비지니스 로직)
- 커질수록 토큰 소모량 증가함
    - 디렉토리별로 클로드.md 를 나눠서 루트 디렉토리 md가 너무 커지지 않도록 하자
- 클로드코드에 md파일에 추가하라고 시키자
- 트리거 키워드로 테스크 자동화 가능
    - 배포 키워드가 입려되면 ..

3. 키보드 단축키 & 네비게이션
- shift tab(plan mode)
    - 파일 쓰기 권한X
    - 검토 요청함
    - 토큰 최적화
    - 실행 정확도
    - 불필요한 코드 생성X
- esc
    - 작업 중단 혹은 새로운 세션
    - 여러 프롬프트중 뒤로 돌아가기
- vscode 배시에서 이미지를 끌어넣어서 보낼수있음
- 패널 분할, 이동 알아두기
- !로 bash 커맨드 실행가능

4. 컨텍스트 관리
- /clear 컨텍스트 초기화
- /context 토큰 사용량 확인
- /compact summary 유지하면서 초기화
- /model 모델 선택
    - 병렬 작업 필요하면 소넷으로?
- /resume 이전 컨벌세이션으로 돌아가기
    - claude --resume
- mcp 사용하는 mcp 서버 확인
- /help, config, export(컨텍스트)
- /output-style 아웃풋 스타일

5. 커스텀 커맨드(/)
- .cluade/command/커맨드명.md에 생김
- 트리거 키워드와 동일
- 세션 종료후 다시 시작해야 인식
