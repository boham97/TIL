# C/C++ 디버깅 플래그 & 메모리 침범 확인 가이드

## 1. 디버그 심볼 (`-g` 계열)

| 플래그 | 설명 |
|---|---|
| `-g` | 기본 디버그 심볼 포함 (가장 많이 사용) |
| `-g0` | 디버그 정보 없음 |
| `-g1` | 최소 정보 (함수명, 외부 변수만) |
| `-g2` | `-g`와 동일, 기본 레벨 |
| `-g3` | 최대 정보 (매크로 정의까지 포함) |
| `-ggdb` | GDB에 최적화된 디버그 포맷 |
| `-ggdb3` | GDB 최적화 + 매크로 정보까지 |
| `-gdwarf-4` | DWARF 버전 지정 (2, 3, 4, 5 가능) |

> `-g3` 또는 `-ggdb3`을 사용하면 `#define` 매크로도 GDB에서 확인 가능

---

## 2. Sanitizer (`-fsanitize` 계열)

| 플래그 | 감지 대상 |
|---|---|
| `-fsanitize=address` | 힙/스택 오버플로, use-after-free, double-free |
| `-fsanitize=undefined` | 정수 오버플로, null 역참조, 잘못된 시프트 등 UB |
| `-fsanitize=leak` | 메모리 누수만 단독 검사 |
| `-fsanitize=thread` | 데이터 레이스 (멀티스레드 경쟁 조건) |
| `-fsanitize=memory` | 초기화되지 않은 메모리 읽기 (Clang 전용) |

> `address`와 `thread`는 **동시 사용 불가**  
> 여러 개를 콤마로 조합 가능: `-fsanitize=address,undefined`

---

## 3. 경고 플래그 (`-W` 계열)

| 플래그 | 설명 |
|---|---|
| `-Wall` | 주요 경고 대부분 활성화 |
| `-Wextra` | Wall에 포함되지 않은 추가 경고 |
| `-Wpedantic` | 엄격한 ISO C 표준 준수 검사 |
| `-Werror` | 모든 경고를 에러로 처리 (빌드 중단) |
| `-Wshadow` | 변수 이름 섀도잉 경고 |
| `-Wconversion` | 암묵적 타입 변환 경고 |
| `-Warray-bounds` | 배열 범위 초과 접근 경고 |
| `-Wformat=2` | printf/scanf 포맷 문자열 불일치 경고 강화 |
| `-Wuninitialized` | 초기화 안 된 변수 사용 경고 |
| `-Wstrict-overflow=2` | 오버플로 기반 최적화 경고 |

---

## 4. 최적화 관련

| 플래그 | 설명 |
|---|---|
| `-O0` | 최적화 없음 (디버깅 시 권장, 변수 추적 정확) |
| `-Og` | 디버깅에 영향 없는 수준의 최적화 |
| `-O2`, `-O3` | 릴리스용 최적화 (디버깅 어려움) |

> 디버깅 시 `-O2` 이상을 쓰면 브레이크포인트가 엉뚱한 줄에 걸리거나 변수가 "optimized out"으로 표시될 수 있음

---

## 5. 기타 유용한 플래그

| 플래그 | 설명 |
|---|---|
| `-fstack-protector-all` | 모든 함수에 스택 카나리 삽입 (스택 오버플로 감지) |
| `-fno-omit-frame-pointer` | 프레임 포인터 유지 (스택 트레이스 정확도 향상) |
| `-D_FORTIFY_SOURCE=2` | printf, memcpy 등 버퍼 오버플로 런타임 검사 |
| `-ftrapv` | 부호 있는 정수 오버플로 시 트랩 발생 |
| `-rdynamic` | 심볼 테이블을 동적 링커에 노출 (backtrace 출력 개선) |

---

## 6. 상황별 권장 조합

### 개발/디버깅 (일반)

```bash
gcc -g3 -O0 -Wall -Wextra -fsanitize=address,undefined \
    -fno-omit-frame-pointer -o myapp main.c
```

### 멀티스레드 디버깅

```bash
gcc -g -O0 -Wall -fsanitize=thread \
    -fno-omit-frame-pointer -o myapp main.c -lpthread
```

### 릴리스

```bash
gcc -O2 -Wall -Wextra -Werror -D_FORTIFY_SOURCE=2 \
    -fstack-protector-all -o myapp main.c
```

## 7. 힙 버퍼 오버플로 디버깅

할당한 메모리 경계를 넘어서 쓰면 힙 메타데이터가 깨지고, `free()` 시점에 크래시가 발생하는 전형적인 패턴이다.

### ASan으로 원인 즉시 찾기 (권장)

```bash
gcc -g3 -O0 -fsanitize=address -fno-omit-frame-pointer -o myapp main.c
./myapp
```

오버플로가 발생하는 **그 순간** 멈추고, 어떤 파일 몇 번째 줄에서 몇 바이트를 초과했는지 정확히 보고한다.

출력 예시:

```

WRITE of size 4 at 0x602000000038 thread T0
    #0 0x40052e in main main.c:12      ← 오버플로 발생 지점
0x602000000038 is located 0 bytes to the right of 40-byte region
allocated by thread T0 here:
    #0 0x7f... in malloc
    #1 0x400501 in main main.c:8       ← 할당 지점
```

### Valgrind으로 확인

```bash
gcc -g -o myapp main.c
valgrind --leak-check=full ./myapp
```

ASan보다 느리고, 경우에 따라 `free()` 시점에서야 보고할 수 있으므로 힙 오버플로 추적에는 ASan이 더 효과적이다.

### ASan vs Valgrind 비교

| 항목 | ASan | Valgrind |
|---|---|---|
| 감지 시점 | 오버플로 발생 즉시 | `free()` 시점에서 보고되기도 함 |
| 실행 속도 | 약 2배 느림 | 약 10~20배 느림 |
| 사용 방법 | 컴파일 플래그 추가 | 별도 도구로 실행 |
| 스택 오버플로 | 감지 가능 | 감지 어려움 |
| 메모리 누수 | `-fsanitize=leak` 별도 사용 | `--leak-check=full`로 상세 보고 |
| 재컴파일 필요 | 예 | 아니오 (`-g`만 있으면 됨) |


예시
```
#include <stdlib.h>

int main(void)
{
    int *arr = malloc(sizeof(int) * 10);  // 10칸
    arr[10] = 42;                          // 11칸 
    free(arr);
    return 0;
}
```

```
gcc -g -fsanitize=address -fno-omit-frame-pointer -o myapp main.c
./myapp
```
